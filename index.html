01

Introduction

Solving the 3D bin-packing problem with robotics and machine learning

Project Goal

TetrisBot automates the challenging task of optimally packing multiple objects into a constrained space. Using a UR robot arm equipped with computer vision and reinforcement learning, our system identifies objects, computes optimal packing sequences, and executes precise pick-and-place operations to stack items like a game of 3D Tetris.

Why It's Interesting

The 3D bin-packing problem is NP-hard, making it computationally intractable for traditional algorithms at scale. Our solution combines learned policies with deterministic placement heuristics, addressing challenges in multi-object perception, grasp planning, motion planning with collision avoidance, and real-time decision making under uncertainty.

Real-World Applications

This technology directly applies to warehouse automation (Amazon-style fulfillment centers), logistics and shipping optimization, automated palletizing in manufacturing, grocery bagging systems, and any domain requiring intelligent object arrangement under space constraints.

02

Design

A two-phase optimization approach decoupling ordering from placement

Design Criteria

Our system needed to reliably detect multiple objects in the workspace, compute space-efficient packing arrangements respecting stability constraints (>50% support), and execute collision-free trajectories to precisely place objects.

Key Design Choices

  • ArUco Markers: Chose marker-based tracking over deep learning detection for reliability and precise 6-DoF pose estimation
  • Internal Grasping: Designed custom hollow boxes allowing the gripper to grasp from inside, eliminating external offset errors
  • Decoupled Planning: Separated packing order (learned via RL) from placement positions (height-map heuristic) for tractability
  • Custom Message Types: Created ROS messages to efficiently communicate box/bin state between perception and planning nodes

Trade-offs

Marker-based tracking requires pre-labeled objects but provides robust, lighting-invariant poses. Internal grasping limits us to hollow objects but ensures consistent grasp quality. The decoupled approach may not find globally optimal solutions but dramatically reduces the search space.

Grasping Strategy

Our 5-step pick-and-place sequence:

  1. Pre-grasp: Position gripper above target box
  2. Grasp: Lower into box interior, open gripper to engage
  3. Orientate: Rotate to target placement orientation
  4. Place: Move to target position, close gripper to release
  5. Home: Return to safe position

ArUco Detection Node

The AR recognition node uses OpenCV's cv2.aruco module to detect markers through an RGB camera, organises into different marker sizes based on ID, and calculates the 3D pose relative to the camera.

AR Tag Identification

From the ArUco node, it retrieves object information for each marker ID, and computes centre offsets for each. It then transforms them to robot's base link frame, and publishes the organised data.

Custom Messages

BoxInfo.msg and BinInfo.msg store object dimensions, IDs, and base_link poses for downstream planning nodes.

Pose Stabilization

To combat noise from ArUco detection, a running average filter is applied to pose estimates, and manual calibration offsets are used for camera-robot alignment. Multiple markers could also be used for redundancy and averaging.

# ar_tag_identify.py - Core perception loop def process_markers(marker_ids, poses): boxes, bin = [], None for id, pose in zip(marker_ids, poses): center = compute_center(pose) base_pose = transform_to_base(center) if id == BIN_ID: bin = BinInfo(id=id, pose=base_pose) else: boxes.append(BoxInfo(id=id, pose=base_pose)) publish_scene(boxes, bin)

RL Policy Network

Deep reinforcement learning planner (DQN/PPO) that learns optimal block placements from grid occupancy and piece geometry, maximizing space utilization while minimizing height and holes.

Learned Reward Structure

Reward function balances line clears, height penalties, hole reduction, and stability bonuses to encourage compact, physically stable packing.

Sim-to-Real Transfer

Policy is trained in a randomized physics simulation and transferred to hardware using domain randomization and targeted real-world fine-tuning.

MoveIt Integration

MoveIt is used for IK trajectory planning together with the custom pose IK planner, collision boxes in the planning scene helps with trajectory planning, and safe execution with velocity and acceleration constraints.

Grasp pipeline computes pre-grasp and grasp poses from perception, evaluates grasp quality, and detects failures for recovery.

Placement Execution

Placement is performed using smooth approach trajectories, controlled descent with force sensing, and post-placement verification for stability.

05

Conclusion

Lessons learned and future directions

How Well Did We Meet Design Criteria?

Our system successfully demonstrated end-to-end autonomous bin packing, detecting objects, computing intelligent packing sequences via RL, and executing precise pick-and-place operations. The decoupled two-phase approach proved effective for managing the complexity of 3D bin packing.

Difficulties Encountered

  • ArUco Noise: Pose estimation was sensitive to lighting and viewing angle, causing inconsistent grasp offsets
  • IK Planner Issues: Collision objects sometimes prevented MoveIt from finding valid paths, requiring dynamic object removal
  • Calibration Drift: Camera-robot transform required frequent manual re-calibration

Known Limitations & Future Work

  • Currently requires pre-labeled objects with ArUco markers
  • Limited to hollow boxes with internal grasping geometry
  • Static workspace‚Äîno dynamic re-planning if objects move

Future Improvements

  • Implement automatic camera calibration
  • Expand to varied box shapes and solid objects
  • Add dynamic object detection via deep learning
  • Support continuous box addition during packing
  • Design custom gripper for external grasps
  • CV-based automatic dimension estimation

Custom Gripper Design

CAD prototype of a custom gripper attachment designed to slide and deform to allow gripper to overshoot the table during placement, and remove offsets to prevent damage to the end effector

Key Takeaway

The project demonstrated that combining learned policies (for ordering) with deterministic heuristics (for placement) is a powerful approach for robotics problems where pure end-to-end learning is intractable.
